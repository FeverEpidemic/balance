## ✨ Features

| Area | Capabilities |
|------|-------------|
| **Auth** | Email/password login, Google OAuth, onboarding flow, profile with theme (light/dark/system) & locale (id/en) |
| **Wallets** | Personal & shared wallets, role-based access (owner, editor, viewer), member invitations via token, up to 5 members per wallet |
| **Transactions** | Income & expense tracking, custom categories per wallet, templates for quick entry, unlimited history |
| **Shared Wallet** | Split bills (equal/custom), settlement tracking for member debts, real-time balance updates |
| **Recurring** | Daily, weekly, monthly recurring transactions — auto-generated by background scheduler |
| **Reports** | Dashboard with income/expense breakdown, category summaries, monthly trends |
| **Budgets** | Monthly budget caps per category, overspend alerts |
| **Savings** | Goal-based savings with progress tracking, saving entry transactions |
| **AI Chat** | Natural language query assistant via DeepSeek API — ask "berapa pengeluaranku bulan ini?" |
| **Payments** | Midtrans payment gateway integration for SaaS subscriptions |
| **API** | Per-user API keys for external integrations, rate-limited via Redis |
| **PWA** | Mobile-responsive design, installable as app |
| **Deployment** | Docker Compose stack (ARM64/AMD64), self-hosted Supabase option, Traefik/Caddy reverse proxy |

## 🚀 Quick Start (Docker)

The fastest way to run Balance:

```bash
# 1. Clone
git clone https://github.com/FeverEpidemic/balance.git
cd balance

# 2. Configure
cp .env.example .env
# Edit .env — at minimum set your Supabase URL and key

# 3. Start
docker compose up --build -d
```

Balance will be available at `http://localhost:3000`.

> **Self-host everything?** Use `docker compose -f docker-compose.self-hosted.yml up --build -d` to spin up a full stack including Supabase, Postgres, Redis, and Mailpit for local email.

**Detailed setup:** See the [Self-Hosted Guide](docs/SELFHOSTED_GUIDE.md) for step-by-step instructions, VPS deployment, custom domains, and troubleshooting.

## 🏠 Self-Hosted Mode

Balance auto-detects whether it's running as **self-hosted** or **SaaS**:

| Mode | How it's detected | Result |
|------|-------------------|--------|
| **Self-hosted** (default) | `SELF_HOSTED_MODE=true` or Midtrans key is missing/placeholder | All features unlocked, no billing checks |
| **SaaS** | `SELF_HOSTED_MODE=false` + valid Midtrans keys | Free tier limited, Premium via Midtrans subscription |

```env
# Force self-hosted mode:
SELF_HOSTED_MODE=true

# Or run as SaaS:
SELF_HOSTED_MODE=false
MIDTRANS_SERVER_KEY=your_real_key
```

The Docker Compose default is `SELF_HOSTED_MODE=true`. Override via `.env`.

## 💻 Local Development

```bash
# Prerequisites
nvm use                     # or Node.js >= 20.9.0
cp .env.example .env        # edit with your credentials

# Install and run
npm install
npm run dev                 # -> http://localhost:3000

# Type-check and test
npm run typecheck
npm run test:unit
```

### Test Wallet Invitation Locally

1. Run the self-hosted stack: `docker compose -f docker-compose.self-hosted.yml up --build`
2. Check email inbox at `http://localhost:8025` (Mailpit)
3. Register a new account, verify email via Mailpit
4. Create an invitation from the wallet members page, share the `/invite/[token]` link

For real SMTP, set `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, and `SMTP_FROM` in `.env`.

### Google Login (Self-Hosted)

```env
GOTRUE_EXTERNAL_GOOGLE_ENABLED=true
GOTRUE_EXTERNAL_GOOGLE_CLIENT_ID=your_google_client_id
GOTRUE_EXTERNAL_GOOGLE_SECRET=your_google_client_secret
```

Add your Supabase Auth callback URL (e.g. `http://localhost:8000/auth/v1/callback`) to Google Cloud OAuth redirect URIs.

Ensure these are set in `.env`:

```env
APP_SITE_URL=http://localhost:3000
GOTRUE_URI_ALLOW_LIST=http://localhost:3000,http://localhost:3000/auth/callback
```

### Redis Cache

Redis is active in the self-hosted stack for dashboard/wallet read caching:

```env
REDIS_URL=redis://redis:6379
REDIS_ENABLED=false          # set true to enable
```

To enable Redis metrics logging:

```env
REDIS_METRICS_ENABLED=true
REDIS_METRICS_INTERVAL_MS=60000
```

The AI Chat endpoint (`/api/chat/*`) also uses Redis for rate limiting (default: 60 requests / 60s per API key). Rate limiting is best-effort — if Redis is unavailable, requests are served without blocking.

## 🐳 Docker Build

### For ARM64 VPS (e.g., Hetzner ARM, Oracle ARM)

Build and start natively on an ARM64 VPS:

```bash
DOCKER_PLATFORM=linux/arm64 docker compose up --build -d
```

Ensure `.env` has the required public values:

```env
NEXT_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=YOUR_PUBLISHABLE_KEY
NEXT_PUBLIC_SITE_URL=https://YOUR_APP_DOMAIN
```

### Build image only

```bash
docker build \
  --platform linux/arm64 \
  --build-arg NEXT_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co \
  --build-arg NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=YOUR_PUBLISHABLE_KEY \
  --build-arg NEXT_PUBLIC_SITE_URL=https://YOUR_APP_DOMAIN \
  -t balance-app:latest .
```

### Multi-arch build and push

```bash
docker buildx build \
  --platform linux/arm64 \
  --build-arg NEXT_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co \
  --build-arg NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=YOUR_PUBLISHABLE_KEY \
  --build-arg NEXT_PUBLIC_SITE_URL=https://YOUR_APP_DOMAIN \
  -t YOUR_DOCKERHUB_USER/balance-app:latest \
  -t YOUR_DOCKERHUB_USER/balance-app:arm64 \
  --push .
```

## 🔧 Environment Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `NEXT_PUBLIC_SUPABASE_URL` | Yes | — | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` | Yes | — | Supabase anon/publishable key |
| `NEXT_PUBLIC_SITE_URL` | Yes | `http://localhost:3000` | Public site URL |
| `SUPABASE_SECRET_KEY` | SaaS only | — | Service role key (server-side only) |
| `SELF_HOSTED_MODE` | No | auto-detect | Force self-hosted (true) or SaaS (false) |
| `MIDTRANS_SERVER_KEY` | SaaS only | — | Midtrans server key |
| `DEEPSEEK_API_KEY` | No | — | Enables AI Chat assistant |
| `SUPPORT_EMAIL` | No | — | Contact email for support links |
| `REDIS_URL` | No | `redis://redis:6379` | Redis connection string |
| `REDIS_ENABLED` | No | `false` | Enable Redis read caching |
| `SMTP_HOST` | Self-hosted | — | SMTP server for auth emails |
| `SMTP_PORT` | Self-hosted | `587` | SMTP port |
| `SMTP_USER` | Self-hosted | — | SMTP username |
| `SMTP_PASS` | Self-hosted | — | SMTP password |
| `SMTP_FROM` | Self-hosted | — | Sender email address |
| `VISION_ENABLED` | No | `true` (if key set) | Enable receipt OCR scanning |
| `VISION_API_KEY` | No | — | API key for Sumopod vision proxy |
| `VISION_BASE_URL` | No | `https://ai.sumopod.com/v1` | Vision AI provider URL |
| `VISION_MODEL` | No | `gemini/gemini-3.1-flash-lite` | Vision model for OCR |
| `VISION_DAILY_LIMIT_FREE` | No | `3` | Free user daily OCR scan limit |
